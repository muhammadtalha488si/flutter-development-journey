## 💻 1. Programming

* **Dart** — the language used to write Flutter apps.

## 🛠️ 2. IDE

* **Android Studio** — main IDE with built-in tools.
* **VS Code** — light and fast, most popular choice.

## 🎨 3. UI

* **Stateless Widget** — never changes (e.g. an icon).
* **Stateful Widget** — can change (e.g. a checkbox).
* **Accessibility** — makes app usable for everyone.
* **Inherited Widget** — shares data down the widget tree (used for Theming & Localization).
* **Material** — Android-style design.
* **Cupertino** — iOS-style design.
* **Assets** — fonts, images, SVGs, audio/video files.

## 🧩 4. Static UI

* **Container, Row, Column, Stack, Expanded** — layout building blocks.
* **Text, Button, Image** — basic things the user sees.

## ⚡ 5. Dynamic UI

* **ListView** — scrollable list.
* **GridView** — scrollable grid.
* **ExpansionTile** — expandable list item.

## 🎬 6. Animations

* **AnimatedWidget / AnimatedBuilder** — build custom animations.
* **AnimationController** — controls an animation's play/duration.
* **CurvedAnimation** — adds easing curves.
* **Hero** — animates a widget between two screens.
* **Opacity** — fades widgets in/out.

## 💾 7. Storage

* **Shared Preferences** — save small settings.
* **File System** — save files on device.
* **SQLite** — local database.

## 📦 8. 3rd Party Libraries

* **font_awesome_flutter** — icon pack.
* **sqflite** — SQLite plugin.
* **flutter_webview_plugin** — show web pages in app.
* **rxdart** — extra Stream tools.
* **bloc_pattern** — BLoC state management.
* **http / dio** — call APIs.
* **get_it** — dependency injection.
* **cached_network_image** — load & cache online images.

## 🔧 9. Behavior Components

* **Media Playback** — play audio/video.
* **Preferences** — save app settings.
* **Permission** — ask for camera, location, etc.
* **Local Notification** — notification from the app itself.
* **Push Notification** — notification from a server.
* **Download Manager** — handle file downloads.

## 🧠 10. State Management

* **setState** — simplest way to update UI.
* **Provider** — share state easily.
* **Redux** — single store for state.
* **BLoC** — state via Streams/events.
* **MobX** — auto-updates UI from observables.

## ✅ 11. Quality Assurance

* **Firebase Crashlytics** — crash reports.
* **Firebase App Distribution** — share test builds.
* **Firebase Analytics** — track user behavior.
* **Google Play Beta / TestFlight / App Center** — beta testing tools.

## 🌿 12. Version Control

* **Git** — track code changes.
* **GitHub / Bitbucket / GitLab** — host your code online.

## 🔥 13. Firebase

* **Firebase Auth** — login system.
* **Firebase Database** — store/sync app data.
* **Firebase Storage** — store files (images, videos).
* **Firebase Messaging** — push notifications.

## 📱 14. Native Integration

**iOS:** Xcode, Swift/Objective-C, Apple Certificates, App Store.

**Android:** Android Studio, Java/Kotlin, App Signing, Google Play Store.
